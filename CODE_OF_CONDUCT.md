# Code of Conduct

## Our Pledge

We as members, contributors, and maintainers of **Vylt** pledge to make participation in this project a respectful, safe, and professional experience for everyone. Vylt is a privacy‑focused, security‑sensitive project, and we value integrity, responsibility, and mutual respect above all.

## Our Standards

Examples of behavior that contribute to a positive environment include:

* Using welcoming and inclusive language
* Being respectful of differing viewpoints and experiences
* Accepting constructive criticism gracefully
* Focusing on what is best for the project and its users
* Acting with honesty, professionalism, and responsibility

Examples of unacceptable behavior include:

* Harassment, abuse, or discrimination of any kind
* Personal attacks, insults, or trolling
* Publishing private information without consent
* Deliberate attempts to weaken security or introduce backdoors
* Any behavior that compromises trust in the project

## Security Responsibility

Vylt deals with encryption and data protection. Contributors are expected to:

* Act in good faith regarding security‑related code
* Never introduce intentional vulnerabilities
* Respect responsible disclosure practices
* Avoid sharing exploit details publicly without maintainer approval

Malicious behavior will result in immediate removal from the project.

## Scope

This Code of Conduct applies within all project spaces, including:

* GitHub repositories
* Issues and pull requests
* Discussions and reviews
* Any official communication channels

## Enforcement

Project maintainers have the right and responsibility to:

* Remove, edit, or reject contributions that violate this Code of Conduct
* Temporarily or permanently ban contributors for unacceptable behavior
* Take any action necessary to protect the project and its users

Decisions made by maintainers are final.
